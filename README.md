# IOC Protocols Models

Protocol definitions, language bindings, and tooling for the IOC L9 / SSTP protocol stack.

## Quick Start

```bash
# Generate all bindings + docs + run tests
make all

# Or step by step:
make generate_bindings        # Python + Go bindings from schema
make generate_docs            # HTML documentation
make test_bindings            # Validate generated code
```

## Project Structure

```
.
├── Makefile                              # All build targets
├── SSTP/
│   ├── spec/l9_schema.json              # Schema (single source of truth)
│   ├── language_bindings/
│   │   ├── python/generated_models.py   # Generated Pydantic v2 models
│   │   ├── golang/generated_models.go   # Generated Go structs
│   │   └── publish_bindings.sh          # Go module publisher (--tag optional)
│   ├── documentation/                   # Published docs + HTML reference
│   └── skills/                          # Hand-authored (not generated)
├── scripts/
│   ├── generate_artifacts.sh            # Full CI pipeline
│   ├── publish_artifacts.sh             # Publish docs + validate bindings
│   └── release_artifacts.sh             # Create git tag + version.json
├── src/                                 # Pydantic source models
└── tests/language_bindings/             # Binding validation tests
```

## Makefile Targets

| Target | Description |
|--------|-------------|
| `make all` | Full pipeline: validate schema, generate bindings + docs, run tests |
| `make generate_bindings` | Generate all language bindings (or `LANGUAGE=python\|golang`) |
| `make generate_docs` | Generate HTML docs from schema |
| `make test_bindings` | Run binding tests (or `LANGUAGE=python\|golang`) |
| `make publish_docs` | Publish documentation artifacts |
| `make publish_bindings` | Validate bindings (use script with `--tag` for git tagging) |
| `make clean` | Remove all generated files |
| `make print-version` | Print current schema version |

## Version Management

**Source of truth:** `SSTP/spec/l9_schema.json` (the `"version"` field)

```bash
# Get current version
make print-version

# Use in scripts
VERSION=$(make -s print-version)
```

## Release Workflow

Run these steps in order for a versioned release:

```bash
VERSION=$(make -s print-version)

./scripts/generate_artifacts.sh          # 1. Generate + validate
./scripts/publish_artifacts.sh "$VERSION" # 2. Publish artifacts
./scripts/release_artifacts.sh "$VERSION" # 3. Create repo git tag
```

Each script accepts an optional version argument. If omitted, it reads from the schema.

## Go Module Publishing

Git tags are **optional** — only needed for external consumers using `go get`.

```bash
# Validate only (no tag)
make publish_bindings LANGUAGE=golang

# Validate + create versioned git tag
./SSTP/language_bindings/publish_bindings.sh golang --tag
```

Tag format: `SSTP/language_bindings/golang/v{version}`

```bash
# Consumer usage
go get github.com/cisco-eti/ioc-cfn-protocols-models/SSTP/language_bindings/golang@v1.0.0

# List published versions
git ls-remote --tags origin | grep "SSTP/language_bindings/golang"
```

Without tagging, local development uses `replace` directives in `go.mod`.

## Skills

Skills live in `SSTP/skills/` and are **hand-authored** — not generated by tooling or LLMs.

## Git Tags (Releases)

Two tags are created during a full release:

| Tag | Purpose |
|-----|---------|
| `{version}` (e.g. `1.0.0`) | Repository-level release tag |
| `SSTP/language_bindings/golang/v{version}` | Go module path-based tag |

The repository tag is created by `scripts/release_artifacts.sh`.
The Go module tag is created by `publish_bindings.sh --tag`.
